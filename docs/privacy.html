本地优先

默认留在本机。

Vibelsland Free 只读取本机已有的 Claude Code、Codex CLI 和 Codex Desktop 状态，用于展示浮岛和处理审批。它不创建账号，不上传遥测，不同步会话内容到远程服务器。

数据边界

读取什么，写入什么。

会读取

  • Claude Code、Codex CLI 和 Codex Desktop 的本机状态。
  • 用于展示活动状态、工具调用、token 摘要和审批请求的本机会话信息。
  • Claude 与 Codex 的本地 hook 配置、会话文件路径和本地运行状态。

会写入

  • 应用配置：~/Library/Application Support/VibelslandFree/config.json。
  • 应用日志：~/Library/Logs/VibelslandFree/app.log。
  • 本地桥接运行文件：~/.vibelsland-free。

保留与删除

本地 token、日志和删除路径说清楚。

本地桥接 token 只用于认证本机 socket；日志用于排障，不会主动上传。卸载 hooks 后，可以手动删除运行目录、配置和日志。

本地桥接 token

  • 运行文件保存在 ~/.vibelsland-free，包含本地 Unix socket 和本地 bridge token。
  • token 只用于本机进程之间确认请求来源，不是线上账号凭证。
  • 删除 ~/.vibelsland-free 会清除运行 token；应用或 hooks 下次启动时会重新建立需要的运行文件。

日志与删除

  • 日志位于 ~/Library/Logs/VibelslandFree/app.log，用于记录连接状态、事件类型、时间戳、本地路径和错误原因。
  • 应用不会主动上传日志；分享日志前请删除敏感路径、提示词、令牌和会话内容。
  • 不再使用时，可以删除 ~/Library/Application Support/VibelslandFree、~/Library/Logs/VibelslandFree 和 ~/.vibelsland-free。

用户控制

Hook 只在你选择时修改。

应用只会在你安装、修复或卸载 hooks 时修改对应配置。Hook payload 会先过滤，只保留会话 ID、事件类型、工作区、审批 ID、工具名、路径和命令等必要元数据。

网络边界

核心功能不需要互联网连接。

Vibelsland Free 的状态显示、来源控制和本地审批桥接都在本机完成。Claude Code、Codex CLI 和 Codex Desktop 可能有各自的网络连接，但它们独立于本应用。

Vibelsland Free 是独立项目，不隶属于 Anthropic、OpenAI、Claude 或 Codex。

官网隐私

访问官网时也保持边界清晰。

这个静态官网不设置自有 cookies，不嵌入分析脚本，也不收集表单。页面由 GitHub Pages 托管；点击 GitHub Release、源码或 Issues 后，会进入 GitHub 的服务边界。

本站不会主动收集

  • 没有自有登录、 newsletter、联系表单或站内追踪脚本。
  • 不会把访问官网时的浏览行为同步回 Vibelsland Free。
  • 不会把下载页访问和应用本地状态关联起来。

外部服务边界

  • GitHub Pages、GitHub Releases 和 GitHub Issues 由 GitHub 提供。
  • 下载、查看源码或提交 issue 时，GitHub 的服务条款和隐私政策适用。
  • 公开 issue 中不要粘贴 token、prompt、会话内容或未脱敏日志。
