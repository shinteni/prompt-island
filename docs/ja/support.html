信頼境界

Gatekeeper と checksum は別の確認です。

v0.1.0 は ad-hoc 署名で、Developer ID notarization ではありません。checksum が通ることは zip が GitHub Release の配布物と一致することを示します。spctl が rejected または not accepted を示す場合、それは notarization の境界であり、SHA-256 検証の失敗とは限りません。

Vibelsland Free は独立したプロジェクトで、Anthropic、OpenAI、Claude、Codex の公式アプリではありません。

状態表示

Claude や Codex の状態が出ない場合。

Claude Code / Codex CLI

  • 設定画面で対象ソースを有効にし、接続のインストールまたは修復を実行します。
  • 新しい Claude Code または Codex CLI セッションを開始し、アイランドとパネルを再確認します。

Codex Desktop

  • Codex Desktop がインストール済みで正常に起動できることを確認します。
  • Vibelsland Free の設定画面で Codex Desktop ソースを有効にします。
  • ヘルスチェックが接続不可を示す場合は、Codex Desktop を開いてタスクを開始してから Vibelsland Free のパネルに戻ります。

承認やツール実行が更新されない

  • 接続を設定した後も Vibelsland Free を起動したままにします。
  • メニューバーの Open Logs を使い、最近の接続エラーや権限エラーを確認します。
  • 関連ツールの設定を手動で変更した場合は、Vibelsland Free の設定画面から接続を修復します。

ローカルデータ

設定とログは Mac 内に保存されます。

Vibelsland Free はローカルツールです。フィードバック時に役立つのは、アプリのバージョン、macOS バージョン、起きたこと、期待した動作、再現手順です。スクリーンショットやログを共有する前に、アカウント名、個人的なパス、会話内容、その他の機密情報を隠してください。

アプリデータの場所 ~/Library/Application Support/VibelslandFree/config.json ~/Library/Logs/VibelslandFree/app.log ~/.vibelsland-free

アンインストール

アプリを削除する前にローカル接続を外します。

  1. 01 ローカル接続を外す

    アプリの設定画面またはメニューバーから Uninstall Hooks を実行します。状態表示のために Vibelsland Free が追加したローカル接続項目を削除します。

  2. 02 終了してアプリを削除

    メニューバーから >_ - island（Vibelsland Free として案内しているアプリ名）を終了し、Applications の >_ - island.app、または Applications に移動したアプリのコピーを削除します。

  3. 03 必要ならローカルデータも削除

    設定やログが不要な場合は、~/Library/Application Support/VibelslandFree、~/Library/Logs/VibelslandFree、~/.vibelsland-free を削除できます。
