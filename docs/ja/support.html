開けない 初回起動と Gatekeeper 状態が出ない Hooks とローカルソース ダウンロード確認 SHA-256 と配布ファイル 削除する Hooks、アプリ、ローカルデータ

信頼境界

Gatekeeper と checksum は別の確認です。

v0.1.0 は ad-hoc 署名で、Developer ID notarization ではありません。checksum が通ることは zip が GitHub Release の配布物と一致することを示します。spctl が rejected または not accepted を示す場合、それは notarization の境界であり、SHA-256 検証の失敗とは限りません。

Vibelsland Free は独立したプロジェクトで、Anthropic、OpenAI、Claude、Codex の公式アプリではありません。

状態表示

Claude や Codex の状態が出ない場合。

Claude Code / Codex CLI

  • 設定画面で対象ソースを有効にし、メニューバーまたは設定画面から Install Hooks / Repair Hooks を実行します。
  • ローカルブリッジが存在するか確認します: ~/.vibelsland-free/bin/vibelsland-bridge。
  • 新しい Claude Code または Codex CLI セッションを開始し、アイランドとパネルを再確認します。

Codex Desktop

  • Codex Desktop がインストール済みで正常に起動できることを確認します。
  • Vibelsland Free の設定画面で Codex Desktop ソースを有効にします。
  • ヘルスチェックが接続不可を示す場合は、Codex Desktop を開いてタスクを開始してから Vibelsland Free のパネルに戻ります。

承認やツール実行が更新されない

  • Hooks を入れた後も Vibelsland Free を起動したままにします。
  • メニューバーの Open Logs を使い、最近の hook、bridge、socket エラーを確認します。
  • Claude や Codex の設定を手動編集した場合は、Repair Hooks でブリッジ設定を再度マージします。

ローカルデータ

トラブルシューティングで使う場所。

Vibelsland Free はローカルツールです。確認するのは主に設定、ログ、ブリッジの実行ディレクトリ、Claude と Codex の hook 設定です。ログを共有する前に、機密性のあるパス、プロンプト、トークン、セッション内容を削除してください。

主なパス ~/Library/Application Support/VibelslandFree/config.json ~/Library/Logs/VibelslandFree/app.log ~/.vibelsland-free ~/.claude/settings.json ~/.codex/hooks.json ~/.codex/config.toml

アンインストール

アプリを削除する前に hooks を外します。

  1. 01 Hooks をアンインストール

    アプリの設定画面またはメニューバーから Uninstall Hooks を実行します。Claude と Codex の hook 設定から Vibelsland のブリッジ項目を削除します。

  2. 02 終了してアプリを削除

    メニューバーから >_ - island（Vibelsland Free として案内しているアプリ名）を終了し、Applications の >_ - island.app、または Applications に移動したアプリのコピーを削除します。

  3. 03 必要ならローカルデータも削除

    設定やログが不要な場合は、~/Library/Application Support/VibelslandFree、~/Library/Logs/VibelslandFree、~/.vibelsland-free を削除できます。

問題報告

再現できる情報を添えてください。

GitHub Issues は公開ページで、通常のバグと伏せ字済みログにだけ使います。脆弱性の悪用手順、token、パス、prompt、完全なログは公開 issue に投稿しないでください。まず GitHub private vulnerability reporting を使い、使えない場合は詳細を含まない最小限の公開 issue で私的連絡先を依頼してください。

Issues GitHub で報告

通常の問題では、Vibelsland Free のバージョン、macOS バージョン、影響を受けるソース、再現手順、伏せ字にしたログ抜粋を含めてください。

 Redaction 公開してはいけない内容を確認

公開 issue には bridge token、作業フォルダの絶対パス、URL query token、AI prompt、セッション全文、個人情報を含むコマンド引数を貼らないでください。

 Privacy プライバシー境界を見る

アプリが読み取り、書き込み、送信しない内容を確認してから、共有するトラブルシューティング情報を決めてください。

 Security セキュリティ報告入口

まず security.txt を確認します。脆弱性の詳細は公開せず、私的報告が使えない場合だけ詳細なしの調整 issue を作成します。
伏せ字の例 /Users/rinka/private-repo => /Users/<user>/<project> https://example.test/callback?token=abc123 => https://example.test/callback?token=<redacted> --api-key sk-... => --api-key <redacted> bridge token / prompt / session body => <redacted summary>
