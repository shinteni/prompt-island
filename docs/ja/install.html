Install & Trust

ダウンロード後の最初の手順。

初回起動の前に、Gatekeeper、ad-hoc 署名、SHA-256、ソース、ローカルデータ境界を一度に確認します。Vibelsland Free のパッケージ、Release ページ、ソースは同じバージョンを軸に確認できます。

初回起動

macOS の確認フローで起動します。

  1. 01 Applications に移動

    Vibelsland-Free-0.1.0-macos.zip を解凍し、>_ - island.app を Applications に移動してから起動します。

  2. 02 Control クリックで開く

    macOS が確認を求めた場合は、Control キーを押しながらアプリのアイコンをクリックし、「開く（Open）」を選びます。この操作で Gatekeeper に入手元を信頼する意思を伝えます。

  3. 03 システム確認を承認

    システムの確認画面でもう一度「開く（Open）」を選びます。この確認はこの Mac に記録され、次回以降は Applications、Spotlight、メニューバーから通常どおり起動できます。

署名の意味

ad-hoc 署名は利用者による信頼確認です。

v0.1.0 は Developer ID notarization なしの配布です。初回起動時は macOS で入手元を確認します。

インストールと状態表示のサポートを見る
ローカル確認 codesign --display --verbose=4 "/Applications/>_ - island.app" spctl --assess --type execute "/Applications/>_ - island.app"

SHA-256

パッケージが置き換えられていないことを確認します。

zip と同名の .sha256 ファイルを同じフォルダに置き、チェックサムを確認します。サイト表示、GitHub Release の checksum ファイル、Mac で計算した結果が一致することを確認してください。

ローカル権限とデータ境界

必要なローカルソースだけを有効にします。

読み取り境界

  • Claude Code、Codex CLI、Codex Desktop の Mac 内の状態を読み取ります。
  • オンラインアカウントを作成せず、テレメトリもアップロードしません。

書き込み境界

  • アプリ設定は ~/Library/Application Support/VibelslandFree/config.json に保存します。
  • アプリログは ~/Library/Logs/VibelslandFree/app.log に保存します。
  • ローカルブリッジのランタイムファイルは ~/.vibelsland-free に置かれます。

利用者の制御

  • 設定画面で Claude Code、Codex CLI、Codex Desktop をソースごとに有効化または無効化できます。
  • アプリが hook 設定を変更するのは、Install、Repair、Uninstall Hooks を選んだときだけです。
  • トラブルシューティング用ログを共有する前に、機密パス、プロンプト、トークン、セッション内容を削除してください。
