安装与信任

下载前最该确认的事。

Vibelsland Free 是免费的吗？

是。v0.1.0 免费下载，源码公开，安装包放在 GitHub Releases。

首次打开为什么需要额外确认？

v0.1.0 采用 ad-hoc 签名，因此首次打开时 macOS 可能要求额外确认。请按住 Control 点按应用并选择“打开（Open）”。

为什么不是 Developer ID notarized？

v0.1.0 是 ad-hoc 签名，不是 Developer ID notarization。Gatekeeper 可能要求 Control-click Open；spctl 显示 rejected 或 not accepted 对当前包是预期结果。SHA-256 只能确认下载文件与 GitHub Release 一致，不能证明开发者身份。

Intel Mac 可以使用吗？

v0.1.0 发布包面向 Apple Silicon / arm64 和 macOS 14+。Intel Mac 不在当前支持范围内。

如何确认下载包没有被替换？

把 zip 和同名 .sha256 文件放在同一目录后，运行 shasum -a 256 -c Vibelsland-Free-0.1.0-macos.zip.sha256 校验。

使用范围

它会读取什么，支持哪些工具。

支持哪些 AI 编程工具？

v0.1.0 支持 Claude Code、Codex CLI 和 Codex Desktop 的本地状态展示、工具调用摘要和审批请求处理。

它会把内容发给 Anthropic 或 OpenAI 吗？

Vibelsland Free 本身不创建账号、不上传遥测、不把会话内容同步到自有服务器。Claude Code、Codex CLI 和 Codex Desktop 各自可能联网，这是它们自己的行为。

和 Anthropic、OpenAI、Claude、Codex 是什么关系？

Vibelsland Free 是独立工具，不隶属、不代表、也没有被 Anthropic、OpenAI、Claude 或 Codex 背书。页面中的产品名只用于说明本地兼容性。

app.log 会保存我的 prompt 吗？

app.log 设计为记录连接状态、事件类型、时间戳、本地路径和错误原因，不把 raw prompt 当日志正文保存。应用会读取本机 Claude/Codex transcript 尾部用于状态显示，所以不能说完全不接触 prompt；分享日志前仍应脱敏 prompt、路径、token 和命令参数。

下载后看到的应用名为什么是“>_ - island.app”？

这是 macOS 应用 bundle 名称。官网、发布包和源码仍使用 Vibelsland Free 作为产品名；这个名称不表示与 Anthropic、OpenAI、Claude 或 Codex 的官方关系。

卸载 hooks 和权限会移除什么？

Uninstall Hooks 会移除 Claude/Codex hook 配置中的 Vibelsland bridge 项。它不会自动删除应用、日志、配置、~/.vibelsland-free、Codex 的 codex_hooks 开关，或第三方工具已经保存的会话级审批状态。
