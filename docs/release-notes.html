v0.1.0

首个公开版本。

这个版本把 Claude Code、Codex CLI 和 Codex Desktop 的本机状态收进一个低干扰的 macOS 浮岛。它适合长任务、工具调用和审批请求频繁出现的 AI 编程工作流，也保留清晰的本地数据边界。

主要功能

v0.1.0 包含的核心能力。

顶部浮岛状态显示

  • 把 AI 编程会话的活动状态、最近工具调用和状态摘要保持在屏幕顶部。
  • 空闲时收起，长任务运行和关键状态出现时展开。
  • 面向 Claude Code、Codex CLI 和 Codex Desktop 的并行工作流。

本地审批处理

  • 在 macOS 原生界面中处理允许一次、本轮允许（支持时）、拒绝和取消任务；Codex CLI Hook 审批只支持允许或拒绝。
  • 保留处理审批所需的基本上下文。
  • 减少在终端、桌面客户端和日志之间切换的频率。

来源与本地连接

  • 按来源启用 Claude Code、Codex CLI 和 Codex Desktop 状态。
  • 通过设置页安装、修复或卸载本地连接。
  • 连接运行文件和日志保留在本机，便于排障和清理。

本地优先边界

  • 无需创建账号，也不上传遥测或远程同步会话内容。
  • 核心状态显示、来源控制和审批处理在本机完成。
  • 隐私页说明了读取、写入和网络边界。

已知边界

v0.1.0 的产品范围。

v0.1.0 聚焦 macOS 14+ 上的本地状态显示和审批处理。Claude Code、Codex CLI 与 Codex Desktop 仍保留各自的账号、网络和会话行为；Vibelsland Free 只整理本机可用状态，并让你按来源控制展示。
