v0.2.1 新增

自定义快捷键、启动亮相与更紧凑的浮岛。

docs/release.json 记录当前发布到 GitHub Releases 的 v0.2.1 包；下载、校验和与 Homebrew cask 均由同一份元数据生成。

快捷键

  • 四个动作都可在设置页录制自定义按键，冲突自动拦截、可恢复默认。
  • 默认空格允许、退格拒绝当前审批；裸键只在有待审批时生效。

启动亮相

  • 每次启动浮岛本体显示约 8 秒，确认应用已运行后自动隐藏。
  • 验证与「减弱动态效果」场景保持原有行为。

更紧凑的浮岛

  • 整体尺寸缩小 20%：药丸 196×34、展开面板 496 宽。
  • 字体与间距等比缩小，版式与信息密度不变。

继承 v0.2.0

  • 一键更新、审批队列、系统通知、本地统计与 Universal 包全部保留。
  • 从 v0.2.0 起可直接在设置页一键升级到本版本。

主要功能

v0.1.0 包含的核心能力。

顶部浮岛状态显示

  • 把 AI 编程会话的活动状态、最近工具调用和状态摘要保持在屏幕顶部。
  • 空闲时收起，长任务运行和关键状态出现时展开。
  • 面向 Claude Code、Codex CLI 和 Codex Desktop 的并行工作流。

本地审批处理

  • 在 macOS 原生界面中处理允许一次、本轮允许（支持时）、拒绝和取消任务；Codex CLI Hook 审批只支持允许或拒绝。
  • 保留处理审批所需的基本上下文。
  • 减少在终端、桌面客户端和日志之间切换的频率。

来源与本地连接

  • 按来源启用 Claude Code、Codex CLI 和 Codex Desktop 状态。
  • 通过设置页安装、修复或卸载本地连接。
  • 连接运行文件和日志保留在本机，便于排障和清理。

本地优先边界

  • 无需创建账号，也不上传遥测或远程同步会话内容。
  • 核心状态显示、来源控制和审批处理在本机完成。
  • 隐私页说明了读取、写入和网络边界。

已知边界

v0.1.0 的产品范围。

v0.1.0 聚焦 macOS 14+ 上的本地状态显示和审批处理。Claude Code、Codex CLI 与 Codex Desktop 仍保留各自的账号、网络和会话行为；Vibelsland Free 只整理本机可用状态，并让你按来源控制展示。
