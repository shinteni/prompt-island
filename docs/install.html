在首次启动前，把 Gatekeeper 确认、ad-hoc 签名、SHA-256 校验、源码和本地数据边界一次核对清楚。Vibelsland Free 的下载包、发布页和源码都围绕同一个版本号对齐。

首次打开

用 macOS 的确认流程启动。

  1. 01 放入 Applications

    解压 Vibelsland-Free-0.1.0-macos.zip，把 >_ - island.app 拖入 Applications，再从 Applications 中启动。

  2. 02 Control 点按打开

    如果 macOS 要求确认，请按住 Control 点按应用图标，选择“打开（Open）”。这个动作明确告诉 Gatekeeper 你信任当前下载来源。

  3. 03 确认系统提示

    在系统提示中再次选择“打开（Open）”。确认会记录在这台 Mac 上，之后可以按常规方式从 Applications、Spotlight 或菜单栏启动。

签名含义

ad-hoc 签名是一次主动信任选择。

v0.1.0 没有 Developer ID notarization，所以首次打开需要你确认来源。

查看安装与状态排障
本机检查 codesign --display --verbose=4 "/Applications/>_ - island.app" spctl --assess --type execute "/Applications/>_ - island.app"

SHA-256

用校验值确认下载包没有被替换。

下载 zip 和同名 .sha256 文件后，把它们放在同一目录执行校验。官网展示值、GitHub Release 的 checksum 文件和本机计算结果应当一致。

打开 v0.1.0 Release

本地权限与数据边界

只启用你需要的本机来源。

读取边界

  • 读取 Claude Code、Codex CLI 和 Codex Desktop 的本机状态。
  • 不创建线上账号，也不上传遥测。

写入边界

  • 应用配置写入 ~/Library/Application Support/VibelslandFree/config.json。
  • 应用日志写入 ~/Library/Logs/VibelslandFree/app.log。
  • 本地桥接运行文件放在 ~/.vibelsland-free。

用户控制

  • 在设置页按来源启用或关闭 Claude Code、Codex CLI 和 Codex Desktop。
  • 只有在你选择 Install、Repair 或 Uninstall Hooks 时，应用才会修改对应 hook 配置。
  • 分享排障日志前，请删除敏感路径、提示词、令牌和会话内容。
