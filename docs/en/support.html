Trust boundary

Gatekeeper and checksum results are different checks.

v0.2.1 is ad-hoc signed, not Developer ID notarized. A passing checksum means the zip matches the GitHub Release file; if spctl reports rejected or not accepted, that is the notarization boundary and does not mean SHA-256 verification failed.

Status display

When Claude or Codex status is missing.

Claude Code / Codex CLI

  • Enable the source in Settings, then run the install or repair connection action.
  • Start a fresh Claude Code or Codex CLI session, then check the island and panel again.

Codex Desktop

  • Confirm Codex Desktop is installed and can run normally.
  • Enable the Codex Desktop source in Vibelsland Free Settings.
  • If the health check says the connection is unavailable, open Codex Desktop, start a task, then return to the Vibelsland Free panel.

Approvals or tool calls do not update

  • Keep Vibelsland Free running after the connection is installed.
  • Use Open Logs from the menu bar and look for recent connection or permission errors.
  • If related tool settings were changed manually, repair the connection from Vibelsland Free Settings.

Local data

Settings and logs stay on your Mac.

Vibelsland Free is a local tool. When sending feedback, the useful details are usually the app version, macOS version, what happened, what you expected, and how to reproduce it. Before sharing screenshots or logs, hide account names, private paths, conversation content, and other sensitive information.

App data locations ~/Library/Application Support/VibelslandFree/config.json ~/Library/Logs/VibelslandFree/app.log ~/.vibelsland-free

Uninstall

Disconnect local integrations before removing the app.

  1. 01 Disconnect local integrations

    Use Uninstall Hooks from the app Settings or menu bar. This removes the local connection entries Vibelsland Free added for status display.

  2. 02 Quit and delete the app

    Quit >_ - island (the app name shown for Vibelsland Free) from the menu bar, then delete /Applications/>_ - island.app, or the app copy you moved into Applications.

  3. 03 Optionally remove local data

    If you no longer need configuration or logs, delete ~/Library/Application Support/VibelslandFree, ~/Library/Logs/VibelslandFree, and ~/.vibelsland-free.
