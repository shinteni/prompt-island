Trust boundary

Gatekeeper and checksum results are different checks.

v0.1.0 is ad-hoc signed, not Developer ID notarized. A passing checksum means the zip matches the GitHub Release file; if spctl reports rejected or not accepted, that is the notarization boundary and does not mean SHA-256 verification failed.

Status display

When Claude or Codex status is missing.

Claude Code / Codex CLI

  • Enable the source in Settings, then use Install Hooks or Repair Hooks from the menu bar or Settings.
  • Confirm the local bridge exists: ~/.vibelsland-free/bin/vibelsland-bridge.
  • Start a fresh Claude Code or Codex CLI session, then check the island and panel again.

Codex Desktop

  • Confirm Codex Desktop is installed and can run normally.
  • Enable the Codex Desktop source in Vibelsland Free Settings.
  • If the health check says the connection is unavailable, open Codex Desktop, start a task, then return to the Vibelsland Free panel.

Approvals or tool calls do not update

  • Keep Vibelsland Free running after hooks are installed.
  • Use Open Logs from the menu bar and look for recent hook, bridge, or socket errors.
  • If Claude or Codex config was edited manually, use Repair Hooks to merge the bridge entries again.

Local data

Locations used during troubleshooting.

Vibelsland Free is a local tool. Troubleshooting usually means checking config, logs, the bridge runtime directory, and Claude or Codex hook files. Before sharing logs, remove sensitive paths, prompts, tokens, and session content.

Common paths ~/Library/Application Support/VibelslandFree/config.json ~/Library/Logs/VibelslandFree/app.log ~/.vibelsland-free ~/.claude/settings.json ~/.codex/hooks.json ~/.codex/config.toml

Uninstall

Remove hooks before removing the app.

  1. 01 Uninstall hooks

    Use Uninstall Hooks from the app Settings or menu bar. This removes Vibelsland bridge entries from Claude and Codex hook configuration.

  2. 02 Quit and delete the app

    Quit >_ - island (the app name shown for Vibelsland Free) from the menu bar, then delete /Applications/>_ - island.app, or the app copy you moved into Applications.

  3. 03 Optionally remove local data

    If you no longer need configuration or logs, delete ~/Library/Application Support/VibelslandFree, ~/Library/Logs/VibelslandFree, and ~/.vibelsland-free.

Report an issue

Include reproducible details.

GitHub Issues are public and should only be used for ordinary bugs with redacted logs. Do not post exploit details, tokens, paths, prompts, or full logs for security vulnerabilities; start with GitHub private vulnerability reporting, or open a minimal public issue asking for private contact if that route is unavailable.

Issues Report on GitHub

For ordinary bugs, include the Vibelsland Free version, macOS version, affected source, reproduction steps, and a redacted log excerpt.

 Redaction Check what must stay private

Do not paste bridge tokens, absolute workspace paths, URL query tokens, AI prompts, full session text, or command arguments containing personal data.
