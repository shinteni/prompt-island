First public release.

This release brings Claude Code, Codex CLI, and Codex Desktop local state into a low-interruption macOS island. It is designed for AI coding workflows where long tasks, tool calls, and approvals need to stay visible without taking over the current window.

Key features

What v0.1.0 includes.

Top-of-screen status island

  • Keeps AI coding activity, recent tool calls, and token summaries at the top of macOS.
  • Contracts while idle and expands for long-running work or important state.
  • Designed around parallel Claude Code, Codex CLI, and Codex Desktop workflows.

Local approval handling

  • Handles allow, deny, continue, and cancel actions in a native macOS surface.
  • Preserves useful context such as approval ID, tool name, path, and command.
  • Reduces switching between terminals, desktop clients, and logs.

Source and hook controls

  • Enables Claude Code, Codex CLI, and Codex Desktop sources independently.
  • Installs, repairs, or uninstalls local hooks from the settings page.
  • Keeps bridge runtime files and logs local for troubleshooting and cleanup.

Local-first boundary

  • No account creation, telemetry upload, or remote session sync from this app.
  • Core status display, source controls, and approval bridging run on the Mac.
  • The privacy page describes read, write, hook payload, and network boundaries.

Known boundaries

The v0.1.0 product scope.

v0.1.0 focuses on local status display and approval bridging on macOS 14+. Claude Code, Codex CLI, and Codex Desktop keep their own account, network, and session behavior; Vibelsland Free organizes the local state available on your Mac and lets you control what appears by source.
