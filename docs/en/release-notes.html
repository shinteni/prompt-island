New in v0.2.1

Customizable hotkeys, a launch presence, and a more compact island.

docs/release.json records the v0.2.1 package currently published on GitHub Releases; the download, checksum, and Homebrew cask are all generated from the same metadata.

Hotkeys

  • Every action can be rebound from settings with a key recorder; conflicts are rejected and defaults restore in one click.
  • Space approves and Backspace declines by default; bare keys are active only while an approval is pending.

Launch presence

  • The island shows itself for about eight seconds on every launch so you can tell the app started.
  • Verification and Reduce Motion behavior stay unchanged.

A more compact island

  • Everything is 20% smaller: the pill is 196×34 and the expanded panel 496 wide.
  • Fonts and spacing shrink proportionally without changing layout or density.

Carried over from v0.2.0

  • One-click updates, the approval queue, notifications, local statistics, and the Universal package all remain.
  • From v0.2.0 you can upgrade to this release directly from settings.

v0.2.1

The current release.

Hotkeys are now customizable per action: Space approves and Backspace declines the pending approval by default, and bare keys are captured only while an approval is pending so normal typing is never affected. The island also shows itself for about eight seconds on every launch, and the whole island is 20% smaller.

v0.1.0

First public release.

This release brings Claude Code, Codex CLI, and Codex Desktop local state into a low-interruption macOS island. It is designed for AI coding workflows where long tasks, tool calls, and approvals need to stay visible without taking over the current window.

Key features

What v0.1.0 includes.

Top-of-screen status island

  • Keeps AI coding activity, recent tool calls, and status summaries at the top of macOS.
  • Contracts while idle and expands for long-running work or important state.
  • Designed around parallel Claude Code, Codex CLI, and Codex Desktop workflows.

Local approval handling

  • Handles accept once, accept for session where supported, decline, and cancel task actions in a native macOS surface; Codex CLI hook approvals support only allow or deny.
  • Preserves the basic context needed to handle approvals.
  • Reduces switching between terminals, desktop clients, and logs.

Sources and local connections

  • Enables Claude Code, Codex CLI, and Codex Desktop sources independently.
  • Installs, repairs, or uninstalls local connections from the settings page.
  • Keeps connection runtime files and logs local for troubleshooting and cleanup.

Local-first boundary

  • No account creation, telemetry upload, or remote session sync from this app.
  • Core status display, source controls, and approval handling run on the Mac.
  • The privacy page describes read, write, and network boundaries.

Known boundaries

The v0.1.0 product scope.

v0.1.0 focuses on local status display and approval handling on macOS 14+. Claude Code, Codex CLI, and Codex Desktop keep their own account, network, and session behavior; Vibelsland Free organizes the local state available on your Mac and lets you control what appears by source.
