First launch

Use the macOS confirmation flow.

  1. 01 Move it to Applications

    Unzip Vibelsland-Free-0.1.0-macos.zip, drag >_ - island.app into Applications, then launch it from Applications.

  2. 02 Control-click Open

    If macOS asks for confirmation, Control-click the app icon and choose Open. That action tells Gatekeeper you trust this download source.

  3. 03 Confirm the system prompt

    Choose Open again in the system prompt. The confirmation is recorded on this Mac, so future launches can use Applications, Spotlight, or the menu bar normally.

Signing meaning

ad-hoc signing is an intentional trust step.

ad-hoc signing gives macOS a code signature on the app bundle, but it does not carry a Developer ID publisher identity. Gatekeeper therefore asks you to confirm the first launch. To decide whether to continue, check the Release page, download asset, SHA-256, and source repository together.

Local permissions and data boundary

Enable only the local sources you need.

Read boundary

  • Reads local Claude Code, Codex CLI, and Codex Desktop state for the island, tool-call summaries, and approval display.
  • Reads local hook configuration, session paths, and runtime state to show whether a source is enabled or needs repair.
  • Creates no online account and does not sync session content to a remote server.

Write boundary

  • App config is written to ~/Library/Application Support/VibelslandFree/config.json.
  • App logs are written to ~/Library/Logs/VibelslandFree/app.log.
  • Local bridge runtime files live in ~/.vibelsland-free.

User control

  • Enable or disable Claude Code, Codex CLI, and Codex Desktop by source in Settings.
  • The app changes hook configuration only when you choose Install, Repair, or Uninstall Hooks.
  • Before sharing troubleshooting logs, remove sensitive paths, prompts, tokens, and session content.
