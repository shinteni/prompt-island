状态显示

Claude、Codex 状态没有出现时。

Claude Code / Codex CLI

  • 在设置页启用对应来源，并使用菜单栏或设置页的 Install Hooks / Repair Hooks。
  • 确认本地桥接文件存在：~/.vibelsland-free/bin/vibelsland-bridge。
  • 重新启动一个 Claude Code 或 Codex CLI 会话，再观察浮岛和面板状态。

Codex Desktop

  • 确认 Codex Desktop 已安装并能正常运行。
  • 在 Vibelsland Free 设置页启用 Codex Desktop 来源。
  • 如果健康检查显示连接不可用，请先打开 Codex Desktop 并启动一个任务，再回到 Vibelsland Free 面板查看。

审批或工具调用不更新

  • 保持 Vibelsland Free 运行，避免只安装 hooks 后退出应用。
  • 打开菜单栏中的 Open Logs，查看最近是否有 hook、bridge 或 socket 错误。
  • 如果曾手动编辑 Claude 或 Codex 配置，优先使用 Repair Hooks 重新合并配置。

本地数据

排障时常用的位置。

Vibelsland Free 是本地工具。排障通常只需要确认配置、日志、桥接运行目录，以及 Claude 与 Codex 的 hook 配置文件。分享日志前请删除敏感路径、提示词、令牌和会话内容。

常用路径 ~/Library/Application Support/VibelslandFree/config.json ~/Library/Logs/VibelslandFree/app.log ~/.vibelsland-free ~/.claude/settings.json ~/.codex/hooks.json ~/.codex/config.toml

卸载

完全移除前，先卸载 hooks。

  1. 01 卸载 hooks

    在应用的设置页或菜单栏执行 Uninstall Hooks。它会从 Claude 和 Codex hook 配置中移除 Vibelsland 的桥接项。

  2. 02 退出并删除应用

    从菜单栏退出 >_ - island（官网产品名 Vibelsland Free），然后删除 Applications 中的 >_ - island.app，或删除你手动放入 Applications 的应用副本。

  3. 03 按需删除本地数据

    如果不再需要历史配置和日志，可以删除 ~/Library/Application Support/VibelslandFree、~/Library/Logs/VibelslandFree 和 ~/.vibelsland-free。
